## Zero-code setup (recommended)

Call `auto_patch()` once at startup — LangSight automatically injects its
callback into every `Crew` and `Agent` you create. No other changes needed.

<Note>
`auto_patch()` reads `LANGSIGHT_URL`, `LANGSIGHT_API_KEY`, and `LANGSIGHT_PROJECT_ID`
from the environment. For best results, load your `.env` file before calling `auto_patch()`.
If env vars are not set at call time, LangSight will lazily initialise the client on the
first span — traces will still be sent as long as the env vars are available by then.
</Note>

```python
# Recommended: load environment variables before auto_patch()
from dotenv import load_dotenv
load_dotenv()  # should come before auto_patch() for best results

# Patch CrewAI
import langsight
langsight.auto_patch()  # zero-code — patches CrewAI automatically

# 3. Your existing CrewAI code unchanged:
from crewai import Agent, Task, Crew

analyst = Agent(role="SQL Analyst", goal="...", backstory="...")
task = Task(description="Analyze revenue", agent=analyst)
crew = Crew(agents=[analyst], tasks=[task])
result = crew.kickoff()
```

Every tool call made by every agent in the crew is automatically traced in
LangSight. The agent's `role` field is used as the `agent_name` on each span
(e.g. "SQL Analyst"), so you see meaningful labels in the dashboard.

### What gets captured automatically

| Signal | How it's captured |
|--------|-------------------|
| Tool calls | Name, input, output, latency, and status for every tool call |
| MCP tool attribution | `mcp__server__tool` names are parsed into `server_name` + `tool_name` breakdown |
| Agent name | Set from the CrewAI agent `role` field |
| Session grouping | Inherited from the active `langsight.session()` context when present |

### Session grouping and prompt/response capture

`auto_patch()` captures tool calls automatically, but **prompt and response
capture, plus session grouping, require a `langsight.session()` wrapper**:

```python
import langsight
langsight.auto_patch()

from crewai import Agent, Task, Crew

analyst = Agent(role="SQL Analyst", goal="...", backstory="...")
task = Task(description="Analyze revenue", agent=analyst)
crew = Crew(agents=[analyst], tasks=[task])

async with langsight.session(input="Analyze Q1 revenue") as session:
    result = crew.kickoff()
    session.set_output(str(result))
```

Without the `session()` wrapper, tool call spans are still captured but are not
grouped into a named session and the human prompt and final response are not
recorded.

## Advanced: manual callback

Use the manual callback when you need per-agent configuration — for example, a
specific `server_name`, a fixed `session_id`, or payload redaction on a single
agent.

```python
from langsight.sdk import LangSightClient
from langsight.integrations.crewai import LangSightCrewAICallback
from crewai import Agent

client = LangSightClient(
    url="http://localhost:8000",
    project_id="my-project",      # scope traces to a project
    redact_payloads=True,          # redact tool args/outputs — recommended if tool args contain PII
)

callback = LangSightCrewAICallback(
    client=client,
    server_name="my-mcp-server",
    agent_name="support-agent",   # optional — overrides role-based auto-detection
)

agent = Agent(
    role="Support Agent",
    tools=[my_mcp_tool],
    callbacks=[callback],          # ← one line added
)
```

Every tool call made by this agent is traced in LangSight.

## Callback methods

The `LangSightCrewAICallback` hooks into three CrewAI events:

| Event | Fires when | Records |
|-------|-----------|---------|
| `on_tool_start` | Tool call begins | Start timestamp, agent name from context |
| `on_tool_end` | Tool call succeeds | Latency + SUCCESS status + server name |
| `on_tool_error` | Tool call fails | Latency + ERROR status + error message |
