## What is LangSight?

LangSight answers the question every AI engineer asks first: "What did my agent call, in what order, how long did each tool take, which ones failed, and what did it cost?"

Langfuse and LangSmith show what your agent *thought* — prompts, completions, evals. LangSight shows what your agent *did* — every tool action, in sequence, with latency, errors, and full cost attribution down to the token.

Agents call three types of things:

1. **MCP servers** (postgres-mcp, jira-mcp, slack-mcp) — via the MCP protocol
2. **Non-MCP tools** (Stripe API, Sendgrid, Python functions, code execution) — direct calls
3. **Sub-agents** (agent-to-agent handoffs in multi-agent workflows)

**LangSight observes all three.** Instrument once at the agent level and you automatically capture everything the agent touched. The `server_name` field in a `ToolCallSpan` can be `"stripe-api"`, `"sendgrid"`, `"openai-api"`, or `"postgres-mcp"` — it is not locked to MCP servers.

### Example: a mixed agent session

```
Agent Session: sess-abc123  (support-agent)
│
├── [MCP]      postgres-mcp / query          42ms  ✓  $0.001
├── [MCP]      jira-mcp / get_issue           89ms  ✓  $0.001
├── [HTTP API] stripe-api / charge_card      340ms  ✓  $0.005
├── [function] calculate_discount              0ms  ✓  $0.000
├── [MCP]      slack-mcp / post_message         —   ✗  timeout
└── [sub-agent] billing-agent
    ├── [MCP]  crm-mcp / update_customer     120ms  ✓  $0.001
    └── [HTTP] sendgrid / send_email          89ms  ✓  $0.002

Total: 680ms | 7 calls | 1 failure | $0.011
```

## What you can observe per tool type

| Tool type | Trace calls | Health check | Security scan | Cost tracking |
|-----------|:-----------:|:------------:|:-------------:|:-------------:|
| MCP servers | Yes | Yes | Yes | Yes |
| HTTP APIs (Stripe, Sendgrid, etc.) | Yes | No | No | Yes |
| Python functions | Yes | No | No | Yes |
| Sub-agents | Yes | No | No | Yes |

MCP servers get extra depth — proactive health checks, CVE scanning (via OSV), 5 of 10 OWASP MCP checks, tool poisoning detection, and schema drift alerts — because the MCP protocol is standard and inspectable. Non-MCP tools appear in every trace but cannot be proactively health-checked.

## LangSight and Langfuse

LangSight is **not** a Langfuse replacement. They answer different questions:

| | Langfuse | LangSight |
|--|----------|-----------|
| Primary question | What did my LLM decide? | What did my agent do? |
| What it traces | LLM calls — prompts, completions, evals | Tool calls — MCP spans, HTTP APIs, sub-agents |
| Cost tracking | LLM token costs | Tool call costs + LLM costs attributed per session |
| Monitoring | Passive only | Proactive MCP health checks, SLO tracking, anomaly detection |
| Security | None | CVE (OSV), 5 of 10 OWASP MCP checks, tool poisoning, auth audit |
| Multi-agent | Agent reasoning chains | Agent-to-agent handoffs via `parent_span_id` |

Use them together. Langfuse shows the LLM decided to call `postgres-mcp/query`. LangSight shows that call took 4 seconds and failed because the connection pool was exhausted.

```
Your agents                LangSight
──────────                 ──────────────────────────────────────
CrewAI          ──SDK──►   POST /api/traces/spans
Pydantic AI     ──SDK──►   POST /api/agents/sessions
LibreChat       ──plugin►  Dashboard → Projects → Sessions
Claude Desktop  ◄──health─ langsight mcp-health
                ◄──scan──  langsight security-scan
                ◄──trace─  langsight sessions
                ◄──rca───  langsight investigate
```

## Installation

```bash
# Install with uv (recommended)
uv tool install langsight

# Or with pip
pip install langsight
```

```bash
langsight init          # auto-discovers your MCP servers and creates .langsight.yaml
langsight sessions      # view recent agent sessions with costs
langsight mcp-health    # check MCP server health
```
